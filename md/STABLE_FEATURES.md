# STABLE FEATURES - 안정화된 기능 목록

이 문서는 완성되어 안정화된 기능들을 기록합니다.
**STABLE** 마커가 있는 기능은 명시적인 개선 요청 없이 수정/삭제할 수 없습니다.

## 규칙

1. **STABLE 기능 추가**: `git push` 전에 완성된 기능에 `@stable` 또는 `STABLE FEATURE` 주석 추가
2. **STABLE 기능 보호**: Pre-commit hook이 자동으로 삭제/수정 감지하여 차단
3. **개선 시에만 수정 가능**: 사용자가 명시적으로 "개선해줘" 요청 시에만 수정 허용

## 마커 사용법

```typescript
// @stable 이미지 드래그앤드롭 순서 조정 (2025-11-13 완성)
// 또는
{/* STABLE FEATURE: 이미지 드래그앤드롭 순서 조정 (2025-11-13 완성) */}
```

## 체크리스트

기능을 STABLE로 표시하기 전 확인:

- [ ] 기능이 완전히 작동하는가?
- [ ] 테스트를 통과했는가?
- [ ] 문서화되었는가? (주석, README 등)
- [ ] Git에 커밋되었는가?
- [ ] `@stable` 또는 `STABLE FEATURE` 주석을 추가했는가?
- [ ] 이 파일(STABLE_FEATURES.md)에 기록했는가?

모두 완료 후 → `git push` 실행

---

## Git Hook 보호

`.git/hooks/pre-commit`이 다음을 자동 체크:
- STABLE/CRITICAL 마커가 있는 코드 삭제 시 차단
- 드래그앤드롭 기능 수정 시 경고
- uploadedImages/Videos 코어 로직 수정 시 경고

**우회 방법**: `git commit --no-verify` (권장하지 않음)
